# Mood Diary

감정 기록을 통해 사용자의 하루를 돌아보고, 감정 흐름을 시각적으로 이해할 수 있도록 돕는 모바일 감정 일기 애플리케이션입니다.

---

## 1. 프로젝트 개요

**Mood Diary**는 사용자가 하루의 감정과 일상을 간단히 기록할 수 있도록 설계된 감정 기반 일기 앱입니다.
텍스트 입력과 감정 선택을 중심으로 감정 변화를 누적·관리하며, 캘린더와 통계를 통해 감정 흐름을 직관적으로 확인할 수 있습니다.

* 개인 감정 기록에 집중한 **프라이빗 저널링 서비스**
* 감정 데이터를 시각화하여 **자기 인식(Self-awareness)** 강화

---

## 2. 주요 기능

### 2.1 일기 작성

* 날짜 자동 설정 (기본값: 오늘)
* 제목 및 본문 텍스트 입력
* 감정 선택 (이모지 / 색상 기반)
* 사진 첨부 (선택 기능)

### 2.2 감정 관리 및 조회

* 월 단위 캘린더 뷰 (날짜별 감정 색상 표시)
* 감정별 필터링 (기쁨, 슬픔 등)
* 일기 수정 / 삭제 기능

### 2.3 감정 시각화

* 감정 변화 추이 그래프
* 기간별 감정 통계 제공

### 2.4 사용자 편의 기능

* 다크 모드 지원
* 비로그인 / 익명 사용 가능
* 로컬 데이터 저장 기반 빠른 접근

---

## 4. 서비스 흐름

```
앱 실행
  ↓
오늘 날짜 표시
  ↓
감정 선택 + 일기 작성
  ↓
로컬 / 서버 저장
  ↓
캘린더 & 그래프에 반영
```

사용자는 복잡한 절차 없이 **감정 선택 → 기록 → 확인**의 흐름으로 자연스럽게 서비스를 이용할 수 있습니다.

---

## 5. UI 구성

* **Home Screen**
  오늘의 날짜, 감정 요약, 최근 일기 카드, 감정 흐름 그래프

* **Diary Write Screen**
  감정 선택 UI + 텍스트 입력 영역

* **Calendar Screen**
  감정 색상 기반 월간 캘린더

* **Filter Screen**
  특정 감정 기준 일기 목록 조회

* **Settings (선택)**
  테마, 데이터 초기화, 백업 기능

---

## 6. 설계 포인트

* 감정 선택을 **텍스트보다 먼저 배치**하여 진입 장벽 최소화
* 감정 데이터를 단순 기록이 아닌 **누적 데이터**로 관리
* UI 요소를 카드 기반으로 구성하여 가독성 확보
* 초기 버전은 로컬 저장 위주 → 확장 시 서버 중심 구조로 전환 가능

## 8. 프로젝트 목적

Mood Diary는 단순한 기록 앱이 아니라,
사용자가 자신의 감정을 **인지하고 이해하는 과정**을 돕는 것을 목표로 합니다.

> "감정을 기록하는 순간, 하루를 돌아볼 수 있다."
